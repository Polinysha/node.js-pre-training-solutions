Task 03: Event Loop Analysis & Async Debugging

Описание реализации:

1. Функции анализа Event Loop:

1.1. analyzeEventLoop()
- Определяет 6 фаз Node.js Event Loop:
  1. Timers - выполнение setTimeout и setInterval
  2. Pending callbacks - обработка I/O коллбэков из предыдущих операций
  3. Idle, prepare - внутренние фазы Node.js
  4. Poll - получение новых I/O событий и выполнение их коллбэков
  5. Check - выполнение setImmediate коллбэков
  6. Close callbacks - выполнение close событий

- Описывает порядок выполнения:
  1. Синхронный код выполняется первым
  2. Микрозадачи process.nextTick
  3. Микрозадачи Promise.then/catch/finally
  4. Макрозадачи (timers, I/O, setImmediate)

- Ключевые правила:
  * Микрозадачи имеют приоритет над макрозадачами
  * process.nextTick имеет высший приоритет среди микрозадач
  * Внутри I/O коллбэков setImmediate выполняется перед setTimeout

1.2. predictExecutionOrder(snippet)
- Анализирует порядок выполнения для двух сниппетов:
  * snippet1: базовый пример с setTimeout, setImmediate, nextTick, Promise
  * snippet2: пример с I/O операцией (fs.readFile)
- Возвращает детальные предсказания с объяснениями

2. Функции исправления асинхронного кода:

2.1. fixRaceCondition()
- Исправляет состояние гонки при параллельной обработке файлов
- Использует Promise.all для параллельного чтения с сохранением порядка
- Автоматически создает отсутствующие файлы
- Возвращает результаты в том же порядке, что и входные файлы

2.2. fixCallbackHell(userId)
- Преобразует вложенные коллбэки в async/await
- Читает три файла параллельно с помощью Promise.all
- Проверяет существование файлов перед чтением
- Обрабатывает ошибки парсинга JSON
- Объединяет данные и сохраняет результат

2.3. fixMixedAsync()
- Исправляет смешанное использование промисов и коллбэков
- Использует последовательные async/await операции
- Создает тестовый файл при его отсутствии
- Обрабатывает ошибки на каждом этапе
